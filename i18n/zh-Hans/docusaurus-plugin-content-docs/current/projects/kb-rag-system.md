# KB RAG 系统 - 完整架构文档

## 项目概览

这是一个全功能的 RAG（检索增强生成）知识库系统，具备智能体编排能力。它结合了语义搜索、实时网页搜索以及 AI 驱动的问答功能。

**核心特性：**
- 📚 **文档索引**：自动将 MDX 文档转换为向量嵌入。
- 🔍 **混合检索**：语义 + 关键词搜索，并配合 Cross-Encoder 重排序。
- 🤖 **智能体编排**：智能路由至专业智能体（知识库、网页搜索、混合模式）。
- 🌐 **网页搜索兜底**：当知识库信息不足时，自动触发网页搜索补充。
- 🛠️ **工具调用**：原生支持 Gemini 函数调用 (Function Calling) API。
- 📖 **引用追踪**：每一个答案都包含准确的源文档引用标注。
- ⚡ **实时响应**：经过极致优化的检索与生成流水线。

## 技术栈

### 后端 (Python)
- **FastAPI**: 高性能 Web 框架。
- **PostgreSQL + pgvector**: 向量数据库存储。
- **Gemini API**: 核心 LLM 推理与 Embedding 生成。
- **Tavily/Brave Search**: 外部网页搜索集成。
- **LangChain**: 文本处理与分块工具。

### 前端 (TypeScript/Docusaurus)
- **Docusaurus**: 文档站点生成器。
- **React**: UI 组件库。
- **SSE (Server-Sent Events)**: 实时流式输出支持。

## 数据流水线

### 阶段 1：文档清洗 (JavaScript)
- **输入**: 原始 MDX 文件。
- **处理**: 移除 React 组件代码、处理特殊语法、生成文档摘要和 Checksum。
- **输出**: 规范化的 JSONL 数据。

### 阶段 2：向量索引 (Python)
- **策略**: 基于标题层级的递归分块 (Recursive Chunking)。
- **向量化**: 调用 Gemini `embedding-001` 模型。
- **增量更新**: 仅在内容发生变更（Checksum 不匹配）时重新构建索引。

## 智能体编排模式

### 1. 知识智能体 (Knowledge Agent)
- **职责**: 针对已知文档库进行精确问答。
- **逻辑**: 执行混合检索 -> 重排序 -> 注入上下文 -> 生成带引用的回答。

### 2. 网页搜索智能体 (Web Search Agent)
- **职责**: 获取实时信息或超出知识库范围的知识。
- **逻辑**: 提取搜索查询 -> 调用 Tavily API -> 总结搜索结果。

### 3. 混合智能体 (Hybrid Agent)
- **职责**: 综合内外部知识。
- **逻辑**: 先尝试 RAG，若置信度不足则补充网页搜索，最后合并生成。

## 部署与配置

### 环境要求
- Python 3.11+
- PostgreSQL 15+ (已安装 pgvector 扩展)
- Doppler CLI (用于安全管理 API 密钥)

### 快速启动
1. **安装依赖**: `cd kb && pip install -e .`
2. **配置变量**: 通过 Doppler 设置 `GEMINI_API_KEY` 和 `DATABASE_URL`。
3. **构建索引**: `python -m kb.cli --stage all`
4. **启动服务**: `uvicorn kb.api.app:create_app --reload`
