## 1. 介绍：什么是 Spring AI 提示词？

Spring AI 的 **Prompt** 系统提供了一种结构化、类型安全的方式来与 AI 模型通信。`Prompt` 类作为组织化的 `Message` 对象和可选 `ChatOptions` 的容器，支持复杂的多轮对话并精确控制模型行为。

### 1.1 Prompt 类

```java
public class Prompt implements ModelRequest<List<Message>> {

    private final List<Message> messages;

    private ChatOptions chatOptions;

    public Prompt(List<Message> messages) {
        this.messages = messages;
    }

    public Prompt(List<Message> messages, ChatOptions chatOptions) {
        this.messages = messages;
        this.chatOptions = chatOptions;
    }

    @Override
    public List<Message> getInstructions() {
        return messages;
    }

    @Override
    public ChatOptions getOptions() {
        return chatOptions;
    }
}
```

`Prompt` 类是向 AI 模型发送请求的主要抽象。它封装了：

- **Messages**：表示对话历史的消息对象序列
- **ChatOptions**：模型特定参数（温度、最大令牌数等）

### 1.2 Message 接口

每个 `Message` 在提示词中体现独特的角色，内容和意图各不相同：

```java
public interface Content {

    String getContent();

    Map<String, Object> getMetadata();
}

public interface Message extends Content {

    MessageType getMessageType();
}
```

对于多模态支持（文本+图像、音频等），消息还可以实现：

```java
public interface MediaContent extends Content {

    Collection<Media> getMedia();
}
```

### 1.3 消息类型和角色

Spring AI 通过 `MessageType` 枚举定义了四种主要消息角色：

```java
public enum MessageType {
    USER("user"),           // 用户的输入 - 问题、命令、陈述
    ASSISTANT("assistant"), // AI 对用户输入的响应
    SYSTEM("system"),       // 指导 AI 行为和风格的指令
    TOOL("tool");           // 工具/函数调用的附加信息
}
```

**角色功能**：

| 角色 | 用途 | 使用方式 |
|------|------|----------|
| **SYSTEM** | 指导 AI 的行为和响应风格 | 在对话前设置参数、规则、上下文 |
| **USER** | 代表用户的输入 | 向 AI 提出问题、命令、陈述 |
| **ASSISTANT** | AI 对用户输入的响应 | 维护对话流程和上下文 |
| **TOOL** | 返回工具调用的信息 | 提供函数执行的数据 |

### 1.4 为什么使用 Spring AI 提示词？

```plaintext
直接 LLM API 调用：                Spring AI 提示词：
┌─────────────────────────────────┐        ┌─────────────────────────────────┐
│ 手动字符串拼接                   │        │ 类型安全的消息 API              │
│ 容易出错的 JSON 处理             │        │ 流畅的 ChatClient API           │
│ 没有模板结构                    │        │ 内置 PromptTemplate             │
│ 手动对话跟踪                    │        │ ChatMemory 集成                │
│ 提供商特定格式                  │        │ 提供商无关的抽象                │
└─────────────────────────────────┘        └─────────────────────────────────┘
         脆弱、重复的代码   →           清晰、可维护的代码
```

**主要优势**：

1. **类型安全**：对消息类型和结构的编译时检查
2. **模板管理**：内置参数化提示词支持
3. **提供商无关**：在 OpenAI、Anthropic、Gemini 之间切换无需代码更改
4. **对话管理**：轻松跟踪多轮对话
5. **多模态支持**：文本、图像、音频和视频的统一 API

## 3. ChatClient.prompt() API - 完整参考

**ChatClient** 提供了构建提示词和调用 AI 模型的流畅 API。它支持同步和流式编程模型。

### 3.1 基本 ChatClient 设置

```java
@Configuration
public class ChatClientConfig {

    @Bean
    public ChatClient chatClient(ChatClient.Builder builder) {
        return builder.build();
    }
}
```

### 3.2 开始提示词：三种方式

```java
@Service
@RequiredArgsConstructor
public class PromptService {

    private final ChatClient chatClient;

    // 方式 1：开始流畅 API（最常见）
    public String promptWithFluentApi(String userInput) {
        return chatClient.prompt()              // 开始流畅 API
            .user(userInput)                    // 添加用户消息
            .call()                             // 执行
            .content();                         // 提取内容
    }

    // 方式 2：使用现有的 Prompt 对象
    public String promptWithPromptObject(Prompt existingPrompt) {
        return chatClient.prompt(existingPrompt) // 使用预构建的 Prompt
            .call()
            .content();
    }

    // 方式 3：使用字符串的便捷方法
    public String promptWithString(String message) {
        return chatClient.prompt(message)        // .user(message) 的简写
            .call()
            .content();
    }
}
```

```java
@GetMapping(value = "/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
public Flux<ServerSentEvent<String>> streamChat(@RequestParam String message) {

    return chatClient.prompt()
        .user(message)
        .stream()
        .content()
        .map(chunk -> ServerSentEvent.<String>builder()
            .data(chunk)
            .build())

            .event("done")
            .data("[DONE]")
            .build()));
```

```java
// 在自定义顾问中访问元数据
@Component
public class MetadataLoggingAdvisor implements CallAroundAdvisor {

    @Override
    public AdvisedResponse aroundCall(
            AdvisedRequest request,
            CallAroundAdvisorChain chain) {

        Map<String, Object> metadata = request.userTextMetadata();
        String userId = (String) metadata.get("userId");

        log.info("Processing request for user: {}", userId);

        return chain.nextAroundCall(request);
    }
}
```

## 5. PromptTemplate 深度解析

**PromptTemplate** 提供一流的提示词管理，支持变量替换、自定义分隔符和资源加载。

### 5.1 基本模板使用

```java
// 创建带占位符的模板
PromptTemplate template = PromptTemplate.builder()
    .template("""
        Translate the following text to {language}.
        Maintain the original tone and style.

        Text: {text}

        Translation:
        """)
    .build();

// 用变量渲染模板
Map<String, Object> variables = Map.of(
    "language", "Spanish",
    "text", "Hello, world!"
);

String renderedPrompt = template.render(variables);
// 输出: "Translate the following text to Spanish. ..."

// 直接创建 Prompt
Prompt prompt = template.create(variables);

// 创建带 ChatOptions 的 Prompt
Prompt promptWithOptions = template.create(
    variables,
    ChatOptions.builder()
        .temperature(0.3)
        .build()
);
```

### 5.2 TemplateRenderer 接口

Spring AI 使用 `TemplateRenderer` 接口进行变量替换：

```java
@FunctionalInterface
public interface TemplateRenderer

    String apply(String template, Map<String, Object> variables);
}
```

**内置渲染器**：

#### 5.2.1 StTemplateRenderer（默认）


#### 5.2.2 NoOpTemplateRenderer

对于没有模板处理的场景：


### 5.6 PromptTemplate 接口

`PromptTemplate` 类实现了多个接口以提供灵活性：

```java
// PromptTemplateStringActions - 渲染为字符串
String rendered = template.render();

// PromptTemplateMessageActions - 创建消息
```

## 8. 提示词选项：控制模型行为

ChatOptions 允许您控制模型参数，如温度、最大令牌数和提供商特定设置。

### 8.1 常见 ChatOptions

```java
// 基本选项

    .model("gpt-4")
    .temperature(0.7)
    .maxTokens(2000)
    .topP(0.9)
    .topK(40)
    .stopSequences(List.of("END", "DONE"))
```

| 选项 | 类型 | 范围 | 描述 |
|------|------|------|------|
| `model` | String | - | 模型标识符（例如 "gpt-4", "claude-3-opus"） |
| `temperature` | Float | 0.0 - 2.0 | 响应的随机性（越低越聚焦） |
| `maxTokens` | Integer | 1 - ∞ | 响应中的最大令牌数 |
| `topP` | Float | 0.0 - 1.0 | 核心采样阈值 |
| `topK` | Integer | 1 - ∞ | Top-k 采样参数 |
| `stopSequences` | List&lt;String&gt; | - | 停止生成的序列 |
| `presencePenalty` | Float | -2.0 - 2.0 | 惩罚新话题 |
| `frequencyPenalty` | Float | -2.0 - 2.0 | 惩罚重复 |

```java
@Component
```

```java
public class ContextEnrichmentAdvisor implements CallAroundAdvisor {

    private final DocumentService documentService;
```

### 10.3 响应格式化顾问

```java
@Component
public class ResponseFormattingAdvisor implements CallAroundAdvisor {

    @Override
    public AdvisedResponse aroundCall(
            AdvisedRequest request,
            CallAroundAdvisorChain chain) {

        // 执行请求
        AdvisedResponse response = chain.nextAroundCall(request);

        // 格式化响应
        String originalContent = response.response()
            .getResult()
            .getOutput()
            .getText();

        String formattedContent = formatResponse(originalContent);

        // 创建带格式化内容的新响应
        ChatResponse formattedResponse = ChatResponse.builder()
            .from(response.response())
            .addGeneration(ChatGeneration.builder()
                .from(response.response().getResult())
                .build(GenericAssistantMessage.from(formattedContent))
                .build())
            .build();

        return AdvisedResponse.from(response)
            .response(formattedResponse)
            .build();
    }

    private String formatResponse(String content) {
        // 如果缺少，添加 markdown 格式
        if (!content.contains("#") && !content.contains("##")) {
            content = "## Response\n\n" + content;
        }

        // 确保代码块有语言标签
        content = content.replaceAll("```\\s*\\n", "```text\n");

        return content;
    }

    @Override
    public int getOrder() {
        return Ordered.LOWEST_PRECEDENCE;  // 最后执行
    }

    @Override
    public String getName() {
        return "ResponseFormattingAdvisor";
    }
}
```

### 10.4 动态提示词顾问

```java
@Component
@RequiredArgsConstructor
public class DynamicPromptAdvisor implements CallAroundAdvisor {

    private final PromptRepository promptRepository;

    @Override
    public AdvisedResponse aroundCall(
            AdvisedRequest request,
            CallAroundAdvisorChain chain) {

        // 从请求确定提示词类型
        String promptType = classifyPrompt(request);

        // 加载动态增强
        List<PromptEnhancement> enhancements =
            promptRepository.findByType(promptType);

        // 构建动态增强文本
        String dynamicEnhancement = enhancements.stream()
            .filter(PromptEnhancement::isActive)
            .map(PromptEnhancement::getContent)
            .collect(Collectors.joining("\n"));

        // 应用增强
        if (!dynamicEnhancement.isEmpty()) {
            AdvisedRequest enhanced = AdvisedRequest.from(request)
                .userText(request.userText() + "\n\n" + dynamicEnhancement)
                .build();

            return chain.nextAroundCall(enhanced);
        }

        return chain.nextAroundCall(request);
    }

    private String classifyPrompt(AdvisedRequest request) {
        String userText = request.userText().toLowerCase();

        if (userText.contains("code") || userText.contains("function")) {
            return "code_generation";
        } else if (userText.contains("explain") || userText.contains("what is")) {
            return "explanation";
        } else if (userText.contains("review") || userText.contains("analyze")) {
            return "analysis";
        } else {
            return "general";
        }
    }

    @Override
    public int getOrder() {
        return Ordered.HIGHEST_PRECEDENCE + 50;
    }

    @Override
    public String getName() {
        return "DynamicPromptAdvisor";
    }
}
```

```java
    public String translateBatch(
            List<String> texts,
            String targetLanguage,
            String context) {

        String batchPrompt = """
            Translate the following texts to {language}.
            
            Context: {context}
            
            {texts}
            
            Provide translations in the same order, numbered:
            1.
            2.
            3.
            ...
            """;

        String numberedTexts = IntStream.range(0, texts.size())
            .mapToObj(i -> String.format("[%d] %s", i + 1, texts.get(i)))
            .collect(Collectors.joining("\n"));

        return translationClient.prompt()
            .user(u -> u.text(batchPrompt)
                .param("language", targetLanguage)
                .param("context", context)
                .param("texts", numberedTexts))
            .call()
            .content();
    }

    private String determineDomain(String context) {
        if (context.toLowerCase().contains("medical") ||
            context.toLowerCase().contains("health")) {
            return "medical and healthcare";
        } else if (context.toLowerCase().contains("legal")) {
            return "legal and compliance";
        } else if (context.toLowerCase().contains("technical") ||
                   context.toLowerCase().contains("software")) {
            return "software and technology";
        } else {
            return "general business";
        }
    }
```
